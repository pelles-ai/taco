CAIP — Agent Composition

End-to-End Construction Workflow

How independently-built AI agents from different companies compose through CAIP's standardized schemas. Each agent's output is the next agent's input — typed, validated, and interoperable.

Every Agent is Opaque
Agents don't expose internals. Proprietary logic, pricing models, and trade secrets stay private. Only the interface matters: Agent Card (metadata) + typed schema I/O. Any CAIP agent works with any other CAIP agent — built by different companies, using different AI models, interoperating because they share a common construction vocabulary.

Standardized Data Schemas
bom-v1 Bill of Materials Complete
  • projectId, trade, csiDivision
  • lineItems[] — qty, unit, material, spec section
  • alternates[] — manufacturer, part numbers
  • metadata — confidence score, source docs
  • Units: EA, LF, SF, CF, CY, TON, LB, GAL, LS
rfi-v1 Request for Information Complete
  • subject, question, category
  • priority — low, medium, high, critical
  • references[] — sheets, grid coords, markup
  • suggestedResolution, relatedBOMItems
  • assignedTo — role, company, contact
estimate-v1 Cost Estimation In Progress
  • Cost data from BOM line items
  • Labor, material, equipment breakdowns
  • Markup and contingency factors
  • Schema definition pending
schedule-v1 Project Schedule In Progress
  • Activities tied to BOM + estimate
  • Dependencies and sequencing
  • Trade coordination windows
  • Schema definition pending
Composable by Design
Any CAIP agent works with any other. A takeoff vendor's output flows to an estimating vendor's input. Different companies, different AI models, same typed schemas.
Human-in-the-Loop
Agents can pause for human review via A2A's input-required state. RFIs route to architects, design decisions wait for approval, then pipelines resume.
Discoverable at Scale
GC orchestrators query the Agent Registry by trade, CSI division, project type, and platform integration. Find the right agent for each phase automatically.
